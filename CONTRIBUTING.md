# Contributing

We appreciate your interest in improving the Omni TrueNAS provider! This project uses an **issues-only** contribution model.

## How to Contribute

### 1. Open an Issue

We do **not** accept pull requests. Instead, open a [GitHub Issue](https://github.com/bearbinary/omni-infra-provider-truenas/issues/new/choose) describing what you'd like — a bug fix, feature, or improvement. Be specific: include the behavior you want, the problem it solves, and any context that helps us understand the request.

Good issues include:
- A clear description of the desired outcome
- Steps to reproduce (for bugs)
- Environment details (TrueNAS version, deployment method, etc.)
- Why it matters to your use case

### 2. Fork for Prototyping (Optional)

You're welcome to fork the repo and prototype a solution. If you do, reference your fork in the issue so we can review your approach and merge it on our side. This helps us move faster while keeping the codebase consistent.

### 3. We Handle the Merge

Once we agree on the approach, we'll implement or merge the change. This lets us maintain code style, test coverage, and release quality.

## Reporting Bugs

Use the [Bug Report](https://github.com/bearbinary/omni-infra-provider-truenas/issues/new?template=bug_report.md) issue template. Include:
- TrueNAS SCALE version
- Provider version / Docker image tag
- Deployment method (TrueNAS app, Kubernetes, Docker Compose)
- Logs (set `LOG_LEVEL=debug` for verbose output)

## Requesting Features

Use the [Feature Request](https://github.com/bearbinary/omni-infra-provider-truenas/issues/new?template=feature_request.md) issue template. Describe what you want and why.

## Code of Conduct

This project follows the [Contributor Covenant](CODE_OF_CONDUCT.md). Be respectful and constructive.
